The enterprise problem

AI coding agents are being introduced into production environments before teams have a serious model for authority, auditability, data movement, plugin trust, or recovery. Most agent platforms optimize for speed. Tribunus optimizes for control.

Control plane, not chat wrapper

Tribunus is built around sessions, tools, approvals, state, receipts, and runtime boundaries.

  • Session state machine — typed transitions, recovery states, event-sourced history
  • Tool runtime — ~200 built-in tools with capability scoping and approval gates
  • Project awareness — agents understand workspace structure and dependencies
  • Execution receipts — every agent action produces a verifiable receipt
  • Multi-agent orchestration — ACP protocol with isolation boundaries

Security model

Five explicit boundaries govern every execution path:

  • Local execution boundary — agent runtime runs on developer hardware; no cloud dependency for core operations
  • Model/provider boundary — each provider connection configurable with explicit endpoint and capability scope
  • Plugin capability boundary — plugins declare required capabilities; runtime enforces the grant
  • Project workspace boundary — agents operate within scoped directories with configurable allow/deny lists
  • Approval levels — mutating operations require per-action or per-session approval

Status: local execution and workspace boundaries implemented. Plugin enforcement experimental. Provider enforcement in progress.

Data handling

Tribunus is designed so local workflows can keep project data local. Remote-provider usage depends on the configured boundary.

  • Project source code processed locally by default
  • Remote API calls go only to configured providers with user-controlled endpoints
  • Execution receipts contain operation metadata, not project content
  • Telemetry is opt-in, limited to aggregate usage statistics
  • Plugins run in-process with declared capability scopes

Audit and evidence

Every agent action, backend decision, and failure state produces a structured receipt.

  • Receipts capture tool calls, model responses, approval decisions, and error states
  • Session history is event-sourced and replayable
  • Evidence corpus on Hugging Face provides independent verifiability
